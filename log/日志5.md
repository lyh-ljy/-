  一、搭建 API 通信层

  之前前端一直用的是 localStorage模拟数据（mock.js），所有增删改查其实都是在浏览器本地打转，根本没跟后端说过话。今天要把这条路打通。

  在 src/api/ 下新建了两个文件：
  request.js 是 axios 实例，核心是配了两个拦截器：

  请求拦截器做的事——前端习惯用下划线命名（customer_name），但后 Java用驼峰（customerName），所以在请求发出去之前自动把字段名转成驼峰。不管 GET 的查询参数还是 POST的请求体，全部统一处理。

  响应拦截器做的事——后端返回的数据 { code, message, data } 三层包裹，拦截器自动把外面那层拆掉只留data。然后 data 里的字段从驼峰转回下划线，保持前端代码里用的都是下划线风格。

  index.js 是所有 API函数的集合，把登录、客户、业务、工单、人员、基础数据、仪表盘的所有接口都封装成了函数。还保留了一个useMock 开关，设成 false 走真实后端，设成 true 走模拟数据，调试的时候切换很方便。

  四、前后端联调遇到的问题

  1. 字段命名不一致

  前端用 customer_name，后端返回 customerName，数据对不上。在 axios 拦截器里加了双向转换：请求出去snake_case →camelCase，响应回来 camelCase →snake_case。这样两端各用各的习惯，互不干扰。

  2. 中文编码问题

  新建含中文的数据时报 400，英文没问题。在 application.properties 加了server.servlet.encoding.charset=UTF-8 解决。

  3. 新建人员报唯一约束冲突

  手机号字段有唯一键，但不填手机号时传的是空字符串 ''，第二个不填的人也报重复。在 Service里把空字符串转成 null 解决。
